Image loader

Plugin for georeferencing and loading images into QGIS.

Images infomation

"Tools","Find images from folder..." will add a row to the images table for every .jpg file in folder and subfolders.

The plugin can also find image details from a 'raster image load' file produced by the distress plotter macro.

project_folder/data is searched for original jpg filenames.

Load GPS data

GPS data is usually in a file named like "project_folder\Hawkeye Exported Data\MFV1_007-rutacd-1.csv".

This will need loading (through "File, open..., Open gps...") before georeferencing.

Georeference

Georeferencing means adding information to an image to specify it's position on a map.

This plugin does this by writing .tif files when Process... georeference selected images is clicked.

The plugin uses a linear referencing system with the GPS data to find image positions.

Position of an image is defined by a start chainage and end chainage.

Corrections

The gps data can be inaccurate. Corrections may be needed to align images with background maps and adjacent runs.

Each run can have a chainage and offset correction.

A positive chainage correction moves images forward.

A positive offset correction moves images to the left side of the vehicle path.

VRT files

Images can effectivly be combined into "virtual raster text" files.

Viewing a VRT file is much faster and less likely to crash QGIS than viewing its images as seperate layers.

"Process,make combined VRT" makes and loads a .vrt file per run and image type.

Loading distress data

Distress data like cracking,rutting and joint faulting is contained in a folder like 'D:\RAF_BENSON\Data\2024-01-08\MFV1_001\Run 1\LCMS Module 1\Results'

The plugin can load this from .acdx files or xml files. The xml files are simply the unzipped .acdx files from hawkeye.

Tools -> View ->... will apply corrections and make a tempuary layer in QGIS.
