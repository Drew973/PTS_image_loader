Image loader

Tool for georeferencing and loading images into QGIS.

Loading image details

"Tools","Find images from folder..." will add a row to the images table for every .jpg file in folder and subfolders.

The tool can also load image details from a 'raster image load' file produced by the distress plotter macro.

project_folder/data is searched for original jpg filenames.

Loading GPS data

GPS data is usually in a file named like "project_folder\Hawkeye Exported Data\MFV1_007-rutacd-1.csv".

This will need loading (through "File, open..., Open gps...") before georeferencing.

Georeferencing

Georeferencing means adding information to an image to specify it's position on a map.

This plugin does this by writing files when Process... georeference selected images is clicked.

The plugin uses a linear referencing system with the GPS data to find image positions.

Position of an image is defined by a start chainage and end chainage.

Corrections

Corrections can be added to adjust image positions in order to align images with maps and adjacent runs.

These can be viewed/edited in the Corrections tab

A correction moves a position within the images to x,y.

Position within images is in terms of chainage,x_offset,y_offset. Offset is difference between point within the image and point at chainage.

Combining files

Images can be merged into "virtual raster text" files. These are small text files containing links to 1 or more original images.

Viewing the VRT file is much faster than viewing its images as seperate layers.

"Process,make combined VRT" makes and loads a VRT per run and image type.
