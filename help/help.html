Image loader

Tool for georeferencing and loading images into QGIS.

Georeferencing

Georeferencing means adding information to an image to specify it's position on a map.

This plugin does this by writing VRT files when Process... georeference selected images is clicked.

A linear referencing system with the GPS data is used to find image positions.

Position of an image is determined by:

  • Start chainage (image_id*5 + correction)
  • End chainage (image_id*5+5 + correction)
  • Offset (0 + correction)

Adding corrections to a run changes the chainages and offset of it's images. A positive Chainage correction moves images forward along the gps line. Offset moves to the left.

Generating image details from folder

"Tools","Find images from folder..." will add a row to the image details table for every .jpg file in folder and subfolders.

Loading from csv/txt

The tool can load image details from a csv or txt file.

The file needs a header. The tool will ignore case and '_' charactors in field names .

A column named filepath is required. This should contain the filepath of the raster image. This can be absolute or relative to the csv/txt.

The following columns will be used if present,otherwise details will be generated from filePath:

  • runID:run
  • imageId
  • fileName:used as layer_name
  • imageType:this and run can be used to generate groups
  • groups

Saving image details

Image details can be saved as a csv or .txt file.

Filepaths will be saved as a relative path (from the folder containing the csv).

Relative paths allow a folder containing the csv and images to be moved without invalidating the file paths.
